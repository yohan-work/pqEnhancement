비개발자를 위한 실전 AI 활용 가이드

🤖 AI 이해하기

AI는 마법 구슬이 아닙니다.
"엄청 똑똑하지만 눈치 없는 신입 인턴"이라고 생각하세요.

  • 명확하게 지시하지 않으면 엉뚱한 답을 줍니다.
  • 한 번에 모든 걸 시키기보다 단계별로 나누어 시키세요.
  • 틀린 정보를 말할 수 있으니(할루시네이션) 반드시 확인이 필요합니다.

✨ 프롬프트 3원칙

1. 페르소나 (Persona)

AI에게 역할을 쥐어주세요. 전문가처럼 생각하고 답변합니다.

"너는 10년 차 전문 마케터야."
"친절한 초등학교 선생님처럼 설명해줘."

2. 맥락 (Context)

배경 상황을 자세히 설명할수록 좋은 답이 나옵니다.

"나는 지금 다이어트 중이고, 탄수화물을 줄이려고 해."
"우리 서비스는 20대 대학생이 주 타겟이야."

3. 출력 형식 (Output Format)

원하는 결과물의 형태를 지정하세요.

"표로 정리해줘."
"블로그 포스팅 말투로 작성해줘."
"3줄 요약해줘."

📋 바로 쓰는 템플릿

기획 아이디어 생성

너는 기획 전문가야.
[주제]에 대한 창의적인 아이디어 10개를 제안해줘.
각 아이디어는 "아이디어 명 - 한 줄 설명 - 기대 효과" 형식으로 작성해줘.

어려운 개념 쉽게 배우기

나는 [분야]에 대해 전혀 모르는 초심자야.
[개념]에 대해 유치원생도 이해할 수 알기 쉽게 비유를 들어서 설명해줘.

💼 직군별 Advanced 템플릿

마케터

SNS 마케팅 (전략적 카피라이팅)

# Role
너는 10년 차 전문 카피라이터이자 SNS 마케팅 전략가야.

# Task
[제품/서비스]를 홍보하기 위한 인스타그램 피드 광고 문구를 작성해줘.

# Constraints
1. 타겟 오디언스: [20대 직장인 여성] (자기계발과 트렌드에 민감함)
2. 톤앤매너: [친근하면서도 세련된, 위트 있는]
3. 길이: 모바일 가독성을 위해 5줄 이내로 끊어쓰기
4. 필수 포함: [핵심 키워드/특장점]

# Steps
1. 먼저 타겟의 페인포인트(Pain Point)를 분석해줘.
2. 그 페인포인트를 해결해줄 수 있는 우리 제품의 USP(Unique Selling Point)를 연결해줘.
3. 이를 바탕으로 3가지 버전(감성형 / 논리형 / 유머형)의 카피를 작성해줘.

# Output Format
- 버전별 구분
- 추천 해시태그 10개

SEO 블로그 포스팅 기획

# Role
너는 SEO 콘텐츠 마케터야. 검색 상위 노출을 목표로 해.

# Task
[주제]에 대한 블로그 포스팅 아웃라인을 작성해줘.

# Strategy
1. **타겟 키워드 분석**: 사람들이 [주제]를 검색할 때 어떤 의도(Search Intent)를 가질지 먼저 추론해줘.
2. **후킹(Hooking)**: 클릭을 유도할 수 있는 매력적인 제목 5가지를 제안해줘. (숫자 활용, 호기심 자극 등)
3. **구조화**: 체류 시간을 늘릴 수 있는 논리적인 글 구조(서론-본론(3문단)-결론)를 잡아줘.

# Output Format
- 검색 의도 분석 결과
- 추천 제목 5선
- 글 구조 (각 문단별 포함해야 할 핵심 내용 및 예상 분량)
세일즈

B2B 콜드 메일 (가치 제안)

# Role
너는 B2B 세일즈 전문가야.

# Task
[타겟 기업/담당자]에게 보낼 콜드 메일 초안을 작성해줘.

# Context
우리의 서비스 [서비스명]은 [타겟 기업]의 [문제가 되는 상황]을 해결해줄 수 있어.

# Guidelines
1. **Subject Line**: 클릭할 수밖에 없는 간결하고 개인화된 제목 (3가지 옵션)
2. **Opening**: 뻔한 인사말 대신, 상대방의 최근 뉴스나 업계 이슈로 시작하여 라포(Rapport) 형성.
3. **Value Prop**: 우리 서비스의 가치를 "기능"이 아닌 "혜택" 중심(Benefit-driven)으로 설명.
4. **Call to Action**: 부담 없이 "커피 한 잔" 또는 "가벼운 줌 미팅"으로 유도.

# Tone
정중하지만 자신감 있는, 프로페셔널한 비즈니스 톤.

고객 거절 방어 (Feel-Felt-Found)

# Situation
고객이 우리 제안을 듣고 "가격이 예산보다 비싸네요"라고 거절한 상황이야.

# Task
이 거절을 극복하고, 가치를 재확인시킬 수 있는 대응 스크립트를 작성해줘.

# Technique (공감-재정의-가치)
1. **Empathy (공감)**: 가격 부담을 충분히 이해한다고 공감해줘.
2. **Reframing (관점 전환)**: 단순 비용이 아니라, [절약되는 시간/늘어나는 매출] 관점에서 "투자"임을 설명해줘.
3. **Value Assertion (가치 강조)**: 타 경쟁사 대비 우리 [서비스]만이 줄 수 있는 [독보적인 장점]을 강조해줘.

# Strategies
- 옵션 1: ROI(투자 대비 효과) 강조형
- 옵션 2: 할부/분할 납부 등 대안 제시형
- 옵션 3: 타협 없는 프리미엄 가치 고수형
기획자 (PM/PO)

PRD (제품 요구사항 정의서) - MECE

# Role
너는 논리적이고 꼼꼼한 시니어 PO(Product Owner)야.

# Task
[기능/아이디어] 구현을 위한 상세 PRD(제품 요구사항 정의서)를 작성해줘.

# Structure (MECE)
1. **Background & Goal (Why)**: 왜 이 기능을 만들어야 하는가? (비즈니스 임팩트, 유저 문제)
2. **Target Audience (Who)**: 누가 이 기능을 언제 쓰는가?
3. **User Stories (What)**: 핵심 유저 스토리 3가지.
4. **Functional Specs (How)**:
    - UI 요소 및 로직 상세 (초기값, 로딩, 에러 상태 포함)
    - 데이터 요구사항 (수집해야 할 데이터)
5. **Edge Cases**: 발생 가능한 예외 상황 및 에러 처리 (네트워크 오류, 데이터 없음 등)

# Constraint
- 개발자, 디자이너가 읽고 바로 작업할 수 있을 정도로 구체적이어야 함.
- 모호한 표현("적당히", "예쁘게") 지양.

유저 스토리 (INVEST 원칙)

# Task
[기능] 개발을 위한 유저 스토리(User Story)와 인수 조건(AC)을 작성해줘.

# Criteria
1. **INVEST 원칙**을 준수할 것 (Independent, Negotiable, Valuable, Estimable, Small, Testable).
2. **Format**:
    - **Story**: "As a [User Type], I want to [Action], so that [Benefit]."
    - **AC (Acceptance Criteria)**: Gherkin 문법(Given/When/Then)으로 작성하여 테스트 가능하도록 명시.

# Scope
주요 케이스(Happy Path)뿐만 아니라, 실패 케이스(Unhappy Path)도 포함해줘.

💡 꿀팁 및 출처

  • 피드백 주기: "이 부분은 좀 더 자세히 설명해줘"라고 대화를 이어가세요.
  • 영어 활용: 가능하다면 영문 질문이 더 퀄리티가 좋을 때가 많습니다. (번역기 활용 추천)
  • 예시 제공: 원하는 스타일의 예시를 하나 던져주면 기가 막히게 따라합니다. (Few-shot prompting)
